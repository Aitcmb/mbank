# `/scaffold` 模板

## `AGENTS.md` 模板

```markdown
# 项目工作规则

## 会话恢复协议
<!-- /clear 或新会话后，按此顺序恢复上下文 —— 不要一次性读全部文件 -->
1. 读取本文件（AGENTS.md）
2. 读取 `mbank/quickref.md` 的 `## 当前焦点`
3. 根据当前焦点的"需要读取"列表，按需加载对应的 `mbank/context/` 文件
4. 读取 `mbank/progress.md` 的 `## 当前状态`（仅顶部）

## 开始编码前必须阅读
1. `mbank/quickref.md`（每次必读——它是上下文路由层）
2. `mbank/progress.md` 的 `## 当前状态`（每次必读，仅顶部 10 行）
3. 其他文件根据 quickref 指引按需读取

## Compact 保留指令
compact 时必须保留以下信息：
- 当前 Phase 和 Step 编号
- 未完成任务列表
- 最近遇到的错误和解决方案
- 当前正在修改的文件列表

## 通用约束
- 单文件不超过 200 行，超出时优先拆分
- 严格遵守 `mbank/tech-stack.md`
- 未经确认不要引入额外第三方包

## TDD 纪律 [标准+]
- Red → Green → Refactor，严禁跳过 Red 阶段直接写实现
- 测试文件与源文件就近放置
- 验证通过前不进入下一步

## 子代理委托规则
当主上下文派发子代理执行 Step 时：
- 子代理接收：Step 全文 + 技术栈约束 + 相关 context 内容（由主上下文直接写入 prompt）
- 子代理不自行读取 mbank/ 文档
- 子代理报告状态：DONE / DONE_WITH_CONCERNS / NEEDS_CONTEXT / BLOCKED
- 子代理完成后，主上下文独立验证结果，再更新 progress/architecture/quickref

## 验证规则
- 不可仅基于代码阅读声称"应该通过"
- 必须运行验证命令并读取完整输出后，才能声称通过
- 子代理声称完成后，主上下文独立验证，不可转发声明

## 文档同步纪律
- **每完成 1 个 Step**：更新 `progress.md` 当前状态 + `quickref.md` 当前焦点
- **每完成 3 个 Step**：全面同步 quickref 关键路径、architecture 组件表、检查 context/ 覆盖率

## 文档维护
- 完成重大功能后更新 `mbank/architecture.md`
- 完成重大功能后更新 `mbank/quickref.md`
- 需要时补充 `mbank/context/*.md`
```

## `mbank/quickref.md` 模板

```markdown
# [项目名称] - 快速参考

> **定位**：每次对话必读的第一个文件。它是上下文路由层——告诉你"现在该读什么、该做什么"。
> 每完成 1 个 Step 更新当前焦点；每完成重大功能更新关键路径。

## 当前焦点
<!-- /clear 后的恢复入口 —— 每完成 1 个 Step 立即更新 -->
**阶段**: [当前 Phase 名称，如 "Phase 0: 项目初始化"]
**当前任务**: [当前 Step 编号和名称，如 "Step 0.1 - 项目脚手架"]
**需要读取**: [当前任务需要加载的 mbank/context/ 文件，没有则填"无"]

## 关键路径
| 功能 | 入口 | 核心文件 | 上下文 |
|------|------|----------|--------|
(待开发时填充)

## 活跃工作流
(记录当前系统的核心操作流程，用简短的步骤链表示)

## 踩坑速览
| 坑 | 模块 | 详情 |
|----|------|------|
(一行摘要 + 指向 mbank/context/ 的链接)
```

**quickref.md 的定位**：
- `architecture.md` = 完整架构文档（组件职责、数据流、设计决策）
- `quickref.md` = 开发者速查卡 + 上下文路由（当前焦点、关键路径、踩坑摘要 → context 指针）
- quickref 只放"打开项目第一分钟需要知道的事"，不重复 architecture 的内容
